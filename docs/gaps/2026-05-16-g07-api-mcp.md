# G07 API And MCP Product Surfaces

Status: Partial; query/graph/control-plane slices are live, complete API/MCP product surface remains blocking

## Requirement

Web UI and MCP are first-class MVP-1 product surfaces. FastAPI should expose corpus registration, indexing jobs, query, evidence, entity, graph, resolver validation, and provider status unless explicitly de-scoped.

MCP should expose agent-facing retrieval and graph tools backed by the same ASIP core.

## Current Evidence

- `apps/api/main.py` exposes FastAPI query, graph, evidence detail, entity explain, corpus add/list/index, durable job list/detail, resolver inspect/list/add/validate, provider settings/Ollama detection, acceptance listing, and `POST /acceptance/run` execution routes.
- `apps/mcp/tools.py` exposes search, graph expansion, evidence detail, entity explain, corpus add/list/index, durable job list/detail, resolver inspect/list/add/validate, provider settings/Ollama detection, acceptance run listing, and `run_acceptance()` execution.
- Query and graph now call `asip.workbench` over the live SQLite store.
- API/MCP tests verify live SQLite query, graph behavior, evidence detail/entity explain over temp DBs, corpus add/list/index/query through temp DBs, durable job lifecycle events, resolver profile add/list/validate through temp DBs, provider settings roundtrip, Ollama detection failure evidence, acceptance artifact listing, and selected acceptance-query execution.
- Acceptance listing supports both historical semantic-edge QA artifacts and the new `asip.acceptance` runner artifacts, including `partial` counts.
- Web BFF `POST /api/workbench/acceptance/run`, FastAPI `POST /acceptance/run`, and MCP `run_acceptance()` can execute selected AQ IDs through the same core runner and return the full `asip.acceptance` payload.
- `apps/api/tests/test_app.py::test_semantic_edges_endpoint_generates_edges_from_live_db` and `apps/mcp/tests/test_tools.py::test_semantic_edges_tool_generates_edges_from_live_db` verify FastAPI `/semantic-edges` and MCP `semantic_edges_generate()` call the same workbench job already used by core/CLI/Web BFF.
- Web BFF query and graph GET routes now accept explicit `dbPath` and no longer call `ensureWorkbenchIndex()` in read-style requests. Playwright API tests verify an explicit empty DB returns an empty query result or isolated seed graph instead of falling back to default indexed data.
- Web BFF `POST /api/workbench/index` now returns the actual custom `dbPath` for isolated index jobs.
- Web BFF `GET /api/workbench/jobs` and `GET /api/workbench/jobs/{id}` expose the same durable job lifecycle data as FastAPI and MCP.
- MCP read tools no longer call `_ensure_index()`. `search_evidence`, `graph_expand`, `run_acceptance`, `evidence_detail`, and `entity_explain` now read explicit missing/empty DBs without creating the DB or indexing default corpora.
- FastAPI `/query`, `/graph`, and `/evidence/{id}` now honor explicit `db_path` without default DB fallback; `/graph` accepts `db_path`.
- Provider/resolver/corpus status/list helpers now avoid schema migration for missing or empty explicit DBs.
- FastAPI now has live Uvicorn smoke tests that serve real HTTP requests without TestClient and prove both direct Uvicorn startup and the root `pnpm dev:api` script can read provider settings without creating a missing explicit DB.
- `apps/mcp/server.py` now registers the full implemented MCP product tool set, not only search/graph/resolver/acceptance listing; a fake FastMCP matrix test verifies every implemented tool function is exposed by the server entrypoint.
- Web BFF `/api/workbench/evidence/{id}` and `/api/workbench/entities/{symbol}` now read through shared core CLI paths, and a Playwright API smoke proves Web BFF and MCP agree for the same SQLite query, evidence row, and entity.

- The current FastAPI/MCP surface is still limited. Evidence/entity endpoints exist, key read-route no-mutation behavior is now covered for query, graph, evidence/entity, acceptance, provider settings, resolver profiles, and corpus listing, FastAPI has live HTTP smoke, MCP server registration exposes the implemented product tools, and Web/MCP query/evidence/entity agreement is covered. Richer resolved-chain explanation UX and optional real MCP package runtime smoke remain missing or incomplete.

## Remaining Gap

API/MCP no longer wrap fixed QA artifacts for query/graph, and acceptance listing/execution can now use the clean acceptance runner. They are still not complete product surfaces.

They do not yet expose a richer resolved-chain explanation UX or an optional real MCP package runtime smoke. The MCP server registration matrix is now covered with a fake FastMCP runtime; the real external MCP runtime remains skipped in this Python environment because the optional package is not installed.

MCP and FastAPI query/graph/evidence/entity/acceptance read paths no longer perform default-corpus auto-indexing, and provider/resolver/corpora status/list helpers no longer migrate explicit missing/empty DBs. The broader API/MCP surface still needs richer explanation UX review and real MCP runtime smoke before closure or explicit deferral.

## Acceptance Criteria

- FastAPI endpoints read the same indexed data used by Web.
- MCP tools read the same indexed data used by Web.
- API and MCP expose query/evidence, explain symbol, resolved chain, graph expansion, resolver inspect/validate, acceptance run/list, corpus/index status, and provider status.
- Live server smoke tests exist for FastAPI and MCP.
- If FastAPI is replaced by Next.js BFF for MVP-1, that de-scope is explicitly accepted by the user in docs.
- Read-only MCP tools do not silently mutate default local state unless the tool name and response make indexing explicit.

- Implemented targeted control-plane tests:

  - FastAPI and MCP persist separate edge/embedding provider settings.
  - FastAPI and MCP detect Ollama against a supplied base URL and return `requested_url` plus structured failure evidence.
  - FastAPI and MCP fetch evidence detail by evidence id and explain an entity with evidence rows, resolved chains, and graph data from a temp SQLite DB.
- Implemented semantic-edge parity tests:
  - FastAPI `POST /semantic-edges` generates graph edges from an indexed temp DB and configured edge provider.

- Implemented runtime/surface tests:
  - FastAPI serves a live Uvicorn HTTP request without TestClient.
  - Root `pnpm dev:api` starts the FastAPI server and serves the same read-only HTTP request.
  - MCP server registration exposes all implemented product tools through the FastMCP entrypoint under a fake runtime.
- Runtime smoke for the real external MCP server when the optional `mcp` package is installed.
- Broader Web/MCP parity tests for graph/entity routes if those become separate MVP product surfaces.
- Test that read-only MCP query can run against a supplied fixture DB without implicit default DB mutation.

## Not Closed Until

API/MCP can answer from the current index and expose the same core product capabilities as the Web workbench or have explicit user-approved deferrals.
